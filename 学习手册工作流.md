> 目标：提供原材料 → 产出完整的线上学习课程

---

## 一、材料准备

你在对应专题文件夹放好以下材料，我来处理剩下的：

```
内部培训/<专题文件夹>/
├── README.md                 ←（可选）课程说明
├── <课程大纲>.docx            ← 框架结构
├── <逐字稿>.docx              ← 完整教学内容
├── 音频/<音频>.mp3            ← 伴读音频
├── 考试/<试卷>.docx           ← 综合考试题目
├── 考试/<答案与解析>.docx     ← 答案解析
├── 学习卡/<卡片1>.png         ← 速记卡片图片
├── PPT/<课件>.pptx            ← 课件（可选）
└── 【图片】*.svg              ← 知识点配图（可选）
```

**表格版：**

| 材料 | 必需？ | 说明 |
|------|--------|------|
| 课程大纲 .docx | ✅ 必需 | 提取内容做独立大纲页面 |
| 逐字稿 .docx | ✅ 必需 | 提取正文、自测题、知识点 |
| 综合考试试卷 .docx | ✅ 必需 | 提取题目做在线考试 |
| 综合考试答案 .docx | ✅ 必需 | 参考答案与解析 |
| 学习卡 .png | ✅ 必需 | 制作学习卡目录页 |
| 音频分段文稿 | ✅ 我来拆分 | 见下方「音频制作流程」 |
| 配图 .svg | ✅ 我来画 | 嵌入学习手册 |

## 二、制作流程（6步）

```
Step 1 ── 我读取所有材料，理解课程结构
Step 2 ── 生成学习手册（HTML + MD）
Step 3 ── 生成综合考试页面（HTML）
Step 4 ── 生成学习卡目录页、大纲页
Step 5 ── 更新学习平台课程列表
Step 6 ── 推送到 GitHub Pages 上线
```

### Step 1：材料读取
我从文件夹读取所有 docx、图片、音频，理解课程主题、模块划分、知识点。

### Step 2：生成学习手册
产出两份：
- **`<课程名>.md`** — Markdown 版，含图文、自测题、课后练习
- **`<课程名>.html`** — HTML 版（最终发布用），含：
  - 顶部固定返回导航
  - 品牌绿色主题（#00d68f）
  - 伴读音频播放器
  - 学习进度指示
  - 学前自测（互动答题+批改）
  - 正文图文混排
  - 课后自测（互动答题+批改）
  - 配套资源区（综合考试、学习卡、大纲、答案解析）
  - 底部返回链接

### Step 3：生成综合考试
- **`考试/综合考试.html`**
  - 60分钟倒计时
  - 双重确认弹窗（防作弊）
  - 选择题、填空题、简答题、案例题
  - 交卷后客观题自动判分
  - 中途退出自动交卷
  - 结果保存在 localStorage

### Step 4：学习卡 + 大纲
- **`学习卡/index.html`** — 卡片缩略图列表 + Lightbox 大图浏览
- **`大纲/index.html`** — 结构化的大纲页面（从 docx 提取）
- 答案解析需校验：考试完成后方可查看

### Step 5：更新学习平台
- **`学习平台.html`**
  - 在 `courses` 数组里添加一门新课程
  - 包含：课程名称、编号、图标、描述、标签
  - 子项URL：学习手册、综合考试、学习卡、大纲

### Step 6：部署上线
```
cd 04-培训认证
git add -A
git commit -m "新增：<课程名>"
git push
```
GitHub Pages 自动部署，约1-2分钟生效。

## 五、音频制作流程

你提供逐字稿后，我帮拆分成适合录音的段落：

### 拆分规则

| 模块 | 时长参考 | 说明 |
|------|---------|------|
| 课程导言 | 3-5分钟 | 整体引入，四个核心问题 |
| 模块一：重新认识XX | 8-12分钟 | 核心知识点讲解 |
| 模块二：XX与慢病 | 8-12分钟 | 因果链、全景地图 |
| 模块三：深度机制 | 8-12分钟 | 接力赛模型/核心机制 |
| 模块四：连锁反应 | 5-8分钟 | 故障传导、修复思路 |
| 小结与预告 | 2-3分钟 | 回顾+下节预告 |

### 产物

对每段我会输出：

```
音频/（课程文件夹）/
├── 01-课程导言.txt       ← 段落文稿（可直接用于录音）
│    时长约4分钟 · 重点：引入、四个问题
├── 02-模块一.txt
│    时长约10分钟 · 重点：组成、功能
├── 03-模块二.txt
│    时长约10分钟 · 重点：慢病因果链
├── ...
└── 合稿.txt              ← 全文合并版（备查）
```

### 使用方式

- 你可以按段落分别录音
- 某一段录坏了只需重录该段
- 录完后把 MP3 文件放回对应的 `音频/` 文件夹
- 我将其嵌入学习手册的音频播放器
